1. Day-1 Configuration

After installation, complete these steps before going operational.

1.1 QShield SOC — Initial Setup

Step 1: Admin Account

Navigate to http://localhost:8080 and complete the first-run setup wizard. Create your admin account with a strong password (min 12 characters, mixed case, numbers, symbols).

Step 2: Configure Data Sources

Go to SIEM → Data Sources. Add your syslog servers, Windows Event collectors, cloud log feeds, and endpoint agents. The platform supports RFC 5424 syslog, CEF, LEEF, and JSON log formats.

Step 3: Enable AI Engine

Navigate to Settings → AI Configuration. Connect your Ollama instance (http://localhost:11434) or configure the embedded TF-IDF RAG engine. Enable threat correlation and NLP analysis.

Step 4: Activate Modules

Go to Settings → Modules. Enable/disable individual modules (SIEM, SOAR, DLP, EDR, XDR) based on your license and deployment needs.

1.2 QShield IDAM — Initial Setup

Step 1: Create Tenant

Navigate to http://localhost:8090. Log in with your admin credentials and create your first tenant organization.

Step 2: Configure Auth Methods

Go to Settings → Authentication. Enable desired methods: Password, OTP, Magic Link, WebAuthn/FIDO2, Social Login. Configure MFA policies per role.

Step 3: Set Up RBAC

Navigate to Access Control → Roles. Create roles (Admin, Manager, User, Auditor) with appropriate permissions. Assign default role for new users.

Step 4: Federation (Optional)

Configure SSO federation with QShield SOC and external IdPs. Set up OIDC clients with proper redirect URIs and scopes.

1.3 QShield AV — Initial Setup

Daemon Configuration

After installing the daemon service, configure monitoring paths and exclusions:

# Configure monitored directories
qshield_daemon.exe --config set monitor.paths "C:\,D:\"
qshield_daemon.exe --config set monitor.exclude "C:\Windows\Temp,*.log"

# Set quarantine vault location
qshield_daemon.exe --config set quarantine.path "C:\QShield\Quarantine"

# Enable auto-update for signature database
qshield_daemon.exe --config set update.auto true
qshield_daemon.exe --config set update.interval 3600

5.3 Key Metrics to Monitor

  • CPU/Memory: SOC should stay under 70% CPU, 4GB heap in production
  • Database connections: Monitor connection pool exhaustion
  • Auth failures/min: Spike indicates brute-force attack (rate limiter should engage)
  • Disk usage: Audit logs and SIEM data grow — plan for 1GB/day at 1000 events/sec
  • AV daemon uptime: Must be 100% for real-time protection

7.2 Backup Commands

# PostgreSQL backup
pg_dump -h localhost -U qshield_admin qshield_soc | gzip > backup_soc_$(date +%Y%m%d).sql.gz
pg_dump -h localhost -U qshield_admin qshield_idam | gzip > backup_idam_$(date +%Y%m%d).sql.gz

# Audit log backup (NEVER delete originals — compliance requirement)
cp -r ~/.qsdpdp-enterprise/data/audit_log* /backup/audit/

# Restore from backup
gunzip -c backup_soc_20260405.sql.gz | psql -h localhost -U qshield_admin qshield_soc
🔴 DPDP Act 2023 Requirement: Audit logs must be retained for a minimum of 7 years. Never delete audit trails. Ensure backups are encrypted and stored in geographically redundant locations.

8. Upgrade Procedures

8.1 Pre-Upgrade Checklist

  • ✓ Take full database backup
  • ✓ Export current configuration
  • ✓ Review release notes for breaking changes
  • ✓ Schedule maintenance window
  • ✓ Notify stakeholders

8.2 Upgrade Process

# 1. Stop the current service
# (Windows) Stop-Service QShieldSOC
# (Linux) sudo systemctl stop qshield-soc

# 2. Backup database
pg_dump -h localhost -U qshield_admin qshield_soc > pre_upgrade_backup.sql

# 3. Replace JAR/EXE with new version
cp QShield-SOC-Platform-2.1.0.jar /opt/qshield/soc/

# 4. Start the new version (auto-migrates database schema)
java -jar QShield-SOC-Platform-2.1.0.jar --spring.profiles.active=prod

# 5. Verify health
curl http://localhost:8080/actuator/health

# 6. Run compliance tests
mvn test -Dtest="VAPTTest,NISTComplianceTest"

8.3 Rollback

# 1. Stop new version
# 2. Restore database from backup
psql -h localhost -U qshield_admin qshield_soc < pre_upgrade_backup.sql
# 3. Restart previous version
java -jar QShield-SOC-Platform-2.0.0.jar --spring.profiles.active=prod
