2. QShield SOC Platform Installation

Option A — Standalone EXE (Recommended for Windows)

No Java required. Includes bundled JRE.

1
Download QShield-SOC-Platform-2.0.0.exe from the Downloads page.
2
Run the executable:
QShield-SOC-Platform-2.0.0.exe
3
Access the dashboard at http://localhost:8080
4
Complete First-Run Setup — create your admin account when prompted.

Option B — JAR Deployment (Cross-Platform)

Requires OpenJDK 17+ installed.

# Linux / macOS
java -jar QShield-SOC-Platform-2.0.0.jar

# Windows PowerShell
java -jar QShield-SOC-Platform-2.0.0.jar

# With custom port
java -jar QShield-SOC-Platform-2.0.0.jar --server.port=9090

Option C — Windows MSI Installer

1
Download QShield-SOC-Setup-2.0.0.msi
2
Double-click the MSI and follow the wizard
3
Select modules: SIEM, SOAR, DLP, EDR, XDR (all selected by default)
4
Configure database — choose embedded H2 (dev) or PostgreSQL (production)
5
Set admin credentials — choose your own admin email and password
6
Click Install — service starts automatically
⚠️ Security: Never use default credentials in production. Always set strong passwords during installation.

Production Configuration

# Launch with production profile (TLS 1.3, PostgreSQL, rate limiting)
java -Xms512m -Xmx2048m \
  -Dspring.profiles.active=prod \
  -Dserver.ssl.key-store=/path/to/keystore.p12 \
  -Dserver.ssl.key-store-password=YOUR_PASSWORD \
  -jar QShield-SOC-Platform-2.0.0.jar

4. QShield Antivirus Installation

CLI Scanner (Portable, No Install)

1
Download qshield_scan.exe — place anywhere on disk
2
Scan a file or directory:
# Scan a single file
qshield_scan.exe --scan C:\Downloads\suspicious_file.exe

# Scan entire directory recursively
qshield_scan.exe --scan C:\Users --recursive

# Scan with auto-quarantine
qshield_scan.exe --scan C:\ --recursive --quarantine

# Save JSON report
qshield_scan.exe --scan D:\Projects --json --report scan_report.json

Real-Time Protection Daemon

1
Download qshield_daemon.exe
2
Install as Windows service (requires Administrator):
# Install service
qshield_daemon.exe --install

# Start real-time protection
qshield_daemon.exe --start

# Check status
qshield_daemon.exe --status

# Stop service
qshield_daemon.exe --stop
💡 Note: The daemon monitors all file system changes in real-time. Detected threats are automatically quarantined to a secure vault.

GUI Installer (Optional)

# Requires Java 17+ installed
java -jar QShieldAV-Setup.jar

Follow the GUI wizard to select components (Scanner, Daemon, AI Engine) and configure auto-start options.

5. Docker Deployment

QShield SOC — Full Stack

# Quick start (single container)
docker pull qshield/soc-platform:2.0.0
docker run -d -p 8080:8080 --name qshield-soc qshield/soc-platform:2.0.0

6. Production Hardening

TLS Certificate Generation

# Generate PKCS12 keystore with 4096-bit RSA
keytool -genkeypair -alias qshield \
  -keyalg RSA -keysize 4096 \
  -validity 365 -storetype PKCS12 \
  -keystore qshield.p12 \
  -dname "CN=soc.yourdomain.com, O=Your Organization, C=IN"

JWT Secret

# Generate 256-bit secret (use different secrets per service!)
openssl rand -base64 32

Environment Variables

export SSL_KEYSTORE_PASSWORD="your-keystore-password"
export JWT_SECRET="$(openssl rand -base64 32)"
export DB_HOST="postgres.internal"
export DB_USERNAME="qshield_app"
export DB_PASSWORD="strong-db-password"
🔴 Critical: Use different JWT secrets for SOC and IDAM. Never reuse secrets across services. Never commit secrets to version control.

7. Post-Installation Verification

Health Checks

# SOC Platform
curl http://localhost:8080/actuator/health
# Expected: {"status":"UP"}

# IDAM Server
curl http://localhost:8090/actuator/health
# Expected: {"status":"UP"}

# AV Scanner
qshield_scan.exe --help
# Expected: Usage info displayed

Security Verification (Production)

# Verify TLS 1.3
openssl s_client -connect localhost:8443 -tls1_3

# Verify security headers
curl -I https://localhost:8443/
# Check for: X-Frame-Options, Content-Security-Policy, HSTS

# Verify rate limiting (should get 429 after threshold)
for i in $(seq 1 12); do
  curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8090/api/v1/auth/login
done

# Verify unauthenticated API access blocked
curl http://localhost:8080/api/v1/dashboard
# Expected: 401 Unauthorized

Run Compliance Tests

# SOC — 466+ tests
cd QS-DPDP-Enterprise && mvn test

# IDAM — 39 tests
cd QS-IDAM && ./mvnw test -pl qs-idam-server

# AV — NIST KAT
cd QSAV/engine/build && ./test_nist_kat

8. Troubleshooting FAQ

Port already in use

# Use a different port
java -jar QShield-SOC-Platform-2.0.0.jar --server.port=9090

Java not found (JAR deployment)

Use the standalone EXE instead — it bundles its own JRE. Or install OpenJDK 21 from adoptium.net.

Database connection failed (production)

# Verify PostgreSQL is running
pg_isready -h localhost -p 5432

# Check connection credentials match application-prod.properties
psql -h localhost -U qshield_app -d qshield_soc -c "SELECT 1"

TLS certificate errors

# Verify keystore is valid
keytool -list -keystore qshield.p12 -storetype PKCS12

# Check certificate CN matches your domain
keytool -list -v -keystore qshield.p12 -storetype PKCS12 | grep "CN="

AV scanner not detecting threats

Ensure the signature database is up to date. Run qshield_scan.exe --update-db to download latest signatures.
